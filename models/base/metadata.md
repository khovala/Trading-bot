# Пояснение к файлу metadata.json

- Файл фиксирует результат стадии обучения базовых моделей.
- `run_id` — идентификатор запуска пайплайна, в рамках которого обучались модели.
- `trained_models` — список идентификаторов моделей, которые успешно обучены и сохранены:
  - `tabular_regression_baseline`
  - `gru_regression_skeleton`
  - `binary_direction_classifier`
  - `multiclass_action_classifier`

Практический смысл:
- Этот файл нужен как быстрый индекс: какие именно базовые модели доступны в `models/base`.
- По нему можно проверить полноту артефактов перед запуском ансамбля и оценкой качества.
